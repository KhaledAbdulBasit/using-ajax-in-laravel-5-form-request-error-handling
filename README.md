<h2>Introduction</h2>
<p>Here is a simple and lightweight method to submit form data via AJAX. This lightweight method will also be able to handle Laravel&#8217;s form request error messages. No additional client side validation plugins will be necessary for error handling. In this tutorial, we will be using Laravel 5 along with the Bootstrap and jQuery libraries.</p>
<h2>What is AJAX?</h2>
<p>AJAX is stands for Asynchronous JavaScript and XML and allows for web apps to send and retrieve data from the server asynchronously without affecting the current web page. AJAX helps improves user experience by not having to fully reload a web page on every request which then reduces bandwidth.</p>

</ol>

<p>First, I selected the form by id and localizing it into a variable labeled form. Then I&#8217;m building up the AJAX by simply grabbing the action and method from the form itself. Then I&#8217;m grabbing the data from the form and serializing it. I have a two callbacks: success and error. In the error callback, if the status is 422 (unprocessable entity), then I&#8217;m going to loop through the response and dynamically display the error messages in our span tag by its class &#8220;name&#8221; and &#8220;-error&#8221;. If the error status is 400 then that means the credentials were incorrect. I&#8217;m just simply going to alert the user. On success, you can handle it however you like. For example, you can redirect the user to their dashboard or something.</p>
